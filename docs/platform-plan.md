# 平台总计划

## 产品定位

Playable AI Studio 的定位是 AI 可玩广告生产平台。平台不让 AI 直接黑盒生成商业级小游戏，而是让 AI 在清晰阶段中完成可验证、可交接、可编辑的产物。

核心目标：

- 把游戏需求、美术参考和效果资料整理成可执行方案。
- 在研发开始前生成视频预演，提前确认玩法表达和美术方向。
- 生成可继续改造成 playable 的原型游戏工程底座。
- 通过 AI 测试和人工试玩形成质量门。
- 对已确认原型发散多条 playable 创意。
- 对选中的创意逐条制作 playable。
- 对确认后的 playable 提供预览和编辑面板。
- 最终导出 HTML playable。

## 总流程

```text
游戏需求 / 效果图 / 效果视频 / 美术参考
  -> AI 策划与 AI 美术整理方案
  -> 研发前视频预演确认
  -> AI 研发生成原型游戏工程底座
  -> AI 测试自测
  -> 人工试玩确认原型游戏
  -> AI 编导发散多条 playable 创意
  -> 人工选择创意并逐条制作 playable
  -> 人工试玩确认单条 playable
  -> 预览 + 编辑面板精调
  -> 自动 QA
  -> 导出 HTML playable
```

## 核心原则

- AI 负责提炼、预演、生成、测试、发散和辅助编辑。
- 人负责关键确认点，包括需求方向、原型游戏、单条 playable 和最终导出版本。
- 工程产物必须参数化，不能只是一次性 demo。
- 不直接自动修改复杂 Cocos 商业工程。
- 可编辑字段由创意方案和工程实现共同声明。
- 长任务不依赖单个聊天窗口记忆，而依赖仓库中的上下文包和提交记录。

## 仓库分工

- `playable-ai-studio`：平台主仓库，放平台方案、流程、设置、任务系统、编辑器和导出器。
- `playable-ai-workspace`：工作仓库，放具体游戏原型、playable 项目、AI 交接上下文和阶段产物。
- `playable-schema`：历史实验仓，保留公开但不作为正式主线。
