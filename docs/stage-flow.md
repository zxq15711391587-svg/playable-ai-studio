# 阶段流程

## 阶段 1：AI 策划与 AI 美术定案

输入：

- 游戏需求。
- 游戏效果图。
- 游戏效果视频。
- 美术参考图。
- 美术需求文档。

输出：

- 游戏原型需求方案。
- 核心玩法规则、胜负条件、用户操作和关卡流程。
- 美术风格方案、资源拆分清单、UI/特效/动画需求。
- 可配置参数清单。
- 研发与测试验收标准。
- 待确认问题。

## 阶段 1.5：研发前视频预演确认

目标是在研发开始前先把玩法表达、画面风格和广告节奏可视化。

规则：

- 先生成分镜，再生成视频。
- 视频拆成多个 2 到 4 秒镜头，避免一次性长视频失控。
- 视频模型负责画面风格、动作、玩法动态和特效。
- 文字、按钮、进度条、手势和 CTA 由程序后期叠加。

输出：

- 视频预演脚本。
- 分镜表。
- 视频生成 prompt。
- UI 叠加配置。
- 预演视频。
- AI 评审报告。
- 人工确认意见。

## 阶段 2：AI 研发原型游戏工程底座

目标是生成一个可运行、可试玩、可继续改造成最终 playable 的原型游戏工程底座。

要求：

- 核心玩法可玩。
- 代码结构清晰。
- 文案、UI、引导、进度、关卡、资源绑定和核心玩法参数配置化。
- 预留后续编辑器读取和修改的字段。

## 阶段 3：AI 测试自测

AI 测试根据需求和验收标准检查：

- 核心规则。
- 操作手感。
- 关卡完成性。
- 视觉目标。
- 明显 bug。
- 移动端适配风险。
- 性能和加载风险。
- 后续 playable 改造风险。

## 阶段 4：人工试玩确认原型游戏

你试玩 AI 团队做出的原型游戏，确认玩法、手感、美术、节奏和参数方向是否可以进入 playable 创意发散。

## 阶段 5：AI 编导 playable 创意发散

AI 编导基于已确认原型生成多条 playable 创意。每条创意必须包含：

- 创意名称。
- 目标人群。
- 核心卖点。
- 前 3 秒钩子。
- 15 到 30 秒节奏。
- 玩家第一步操作。
- 爽点、卡点和 CTA 时机。
- 画面表现和文案方案。
- 玩法参数、UI 参数和美术资源改动。
- 可编辑字段列表。
- 制作难度和预期转化逻辑。

## 阶段 6：逐条制作 playable

对人工选择的创意逐条制作 playable。一次只制作一条创意，先生成可运行版本，再整理可编辑字段初稿。

## 阶段 7：人工试玩确认单条 playable

你体验单条 playable。确认 OK 后才进入编辑阶段；不 OK 则退回 AI 研发修改。

## 阶段 8：预览与编辑阶段

对已确认 playable 生成实时预览和编辑面板。编辑器只开放该 playable 已声明的字段，不允许随意改工程代码。

## 阶段 9：自动 QA 与 HTML 导出

导出前检查：

- 是否能正常打开。
- 首屏是否可交互。
- CTA 是否可点击。
- 横竖屏是否正确。
- 资源是否缺失。
- 包体是否超限。
- 是否有外链风险。
- 是否符合目标渠道要求。
- 是否存在控制台报错。
- 是否支持移动端尺寸。

输出：

- 单文件 HTML 或 HTML zip 包。
- 录屏视频。
- 版本配置 JSON。
- QA 报告。
